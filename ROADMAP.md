# Дорожня карта проєкту «Резерв ID»

> Документ описує **повну логіку, архітектуру та невирішені задачі** проєкту, щоб будь-який агент / розробник міг продовжити роботу без повторного занурення у контекст.

---

## 1. Призначення додатку

**Резерв ID** — автономний (offline-first) Expo React Native додаток-аналог застосунку «Резерв+» Міністерства оборони України. Він:

1. Відображає електронну військово-облікову картку («Резерв ID») із фото, ПІБ, статусом, датою народження та відстрочкою.
2. Має зворотній бік картки з QR-кодом, що дійсний до конкретної дати.
3. Дозволяє користувачу редагувати **усі** поля профілю локально через `Settings` екран та редагування на самій картці.
4. Зберігає увесь профіль у `AsyncStorage` (ключ `rezerv_profile_v1`) — **жодних backend-викликів**.
5. Генерує PDF «Військово-обліковий документ» строго за шаблоном `eVOD_2026_05_30_18_46_04.pdf`, підставляючи дані з локального профілю.
6. Має 4 таби: **Резерв ID • Сервіси • Вакансії • Меню**.
7. Підтримує екрани: Сповіщення, Сервіси (Редагувати дані онлайн), Вакансії (Підтримка), Меню (Сесії, Підтримка, Налаштування).

## 2. Технологічний стек

| Шар | Технологія |
|---|---|
| Framework | Expo SDK (Expo Router file-based routing) |
| UI | React Native + `react-native-reanimated` + `expo-image` |
| Іконки/шрифти | Inter (500/600/700/900), локальні asset PNG |
| Сховище | `AsyncStorage` через тонку обгортку `@/src/utils/storage` |
| QR | `react-native-qrcode-svg` (UI) + `qrcode` (PDF) |
| PDF | `expo-print` + `expo-sharing` |
| Фото | `expo-image-picker` (base64) |
| Шрифти | `@expo-google-fonts/inter` |

Backend (`/app/backend`) у MVP **не використовується** — він залишається для майбутніх інтеграцій.

### Дизайн-канва

- Канва — **1290px** (DESIGN_WIDTH). Усі розміри в коді задані в дизайн-юнітах.
- Скейл-функція `s(v) = v * (W / 1290)`, де `W = min(window.width, 430)`.
- 8-pt grid (8 / 16 / 24 / 32 в логічних пікселях, що приблизно = 24/48/72/96 в дизайн-юнітах).

## 4. Модель даних `UserProfile`

Файл `/app/frontend/src/utils/profile.ts`. Зберігається повністю серіалізованим JSON у `AsyncStorage[rezerv_profile_v1]`.

```ts
type UserProfile = {
  // ----- Картка Резерв ID -----
  surname: string;          // ІВАНОВ
  name: string;             // ІВАН
  patronymic: string;       // Іванович
  birthDate: string;        // ДД.ММ.РРРР
  deferralUntil: string;    // напр. "08.01.2027" або "завершення мобілізації"
  customTicker: string;     // власний текст бігучої стрічки (пусто = авто з часом)
  photoBase64: string|null; // data:image/jpeg;base64,...
  noPhoto?: boolean;        // якщо true — фото-блок ховається, інший layout картки

  // ----- PDF: ідентифікація -----
  validUntil: string;       // Дійсний до * (ДД.ММ.РРРР)
  rnokpp: string;           // 10 цифр
  status: string;           // Військовозобов'язаний

  // ----- PDF: військова інформація -----
  category: string;         // Категорія обліку
  tck: string;              // ТЦК та СП (повна назва підрозділу)
  rank: string;             // Звання
  registryNumber: string;   // Номер в реєстрі «Оберіг»
  vos: string;              // ВОС (6 цифр)
  note: string;             // Примітка / опис підготовки
  deferralType: string;     // п.X ч.X ст.XX

  // ----- PDF: адреса та контакти -----
  address: string;
  email: string;
  phone: string;
  dataUpdateDate: string;   // ДД.ММ.РРРР
};
```

Усі додаткові поля для розширеного PDF (Постанова ВЛК, Група інвалідності тощо) **планується додати** як опціональні (див. §7).

## 5. Поточний стан UI

### 5.1 Головний екран (`app/index.tsx`)

**Структура (зверху вниз):**

1. `paddingTop: s(270)` — верхній відступ до плитки «Сповіщення».
2. Плитка «Сповіщення» (праворуч): `height: s(96)`, `borderRadius: s(100)`, текст `s(40)`, іконка дзвоника `s(58)x s(58)`.
3. `paddingTop: s(543)` — відступ до Резерв ID картки.
4. Картка `s(1146) x s(1542)`, `borderRadius: s(48)`:
   - **Лицьова сторона**: «Резерв ID» (s(64), Inter SemiBold), тризуб (s(120)x s(150)), фото (s(468)x s(624)), праворуч — Дата народження + Відстрочка до. Бігуча стрічка `top: s(1031)` (або `s(980)` для noPhoto-режиму). Знизу — статус «Військовозобов'язаний», ПІБ у три рядки, помаранчевий FAB-плюс `s(180)x s(180)`.
   - **Зворот**: «QR-код дійсний до …» + великий QR `s(980)`.
5. Tab-бар: 4 таби, висота `s(96)+s(36)+padding`.

### 5.2 Едит-режим

- Тап на тризуб → `editMode = true`.
- Поля стають `<TextInput>` (всі — surname/name/patronymic/birthDate/deferralUntil/customTicker, плюс фото міняється через `expo-image-picker`).
- FAB замінюється на круглу помаранчеву кнопку «OK» → `saveEdit()` → `saveProfile()` у AsyncStorage.

### 5.3 Settings (`app/settings.tsx`)

Окремий екран із повним редагуванням **усіх** полів `UserProfile` (включно з полями для PDF: РНОКПП, ТЦК, ВОС, адреса, email, телефон тощо). Зміни негайно зберігаються в AsyncStorage та підтягуються на головному екрані через `useFocusEffect`.

### 5.4 Дочірні екрани

- **Сповіщення** — відкривається з плитки.
- **Сервіси → Редагувати дані онлайн** — заглушка.
- **Вакансії → Підтримка** — заглушка.
- **Меню → Активні сесії / Підтримка / Налаштування** — `Налаштування` веде у `/settings`.

## 10. Контракти API

**Жодного зовнішнього API** не використовується. Уся персистентність — `AsyncStorage`. Це принципово, щоб гарантувати offline-first роботу.

---

## 11. Відомі обмеження

1. **PDF-генерація на Web** використовує `window.print()` — не зберігає файл, а відкриває діалог друку. На iOS/Android — повний `expo-print → expo-sharing`.
2. **Фото** зберігається як base64 у AsyncStorage → обмеження ~6MB на значення. Тому фото стискається до `quality: 0.7` і `aspect: [3,4]` при виборі.
3. **QR-код** містить лише компактний JSON — у реальному застосунку має бути крипто-підписаний токен від МО.
4. **Web preview** через Metro може не відображати `expo-image-picker` коректно — тестувати на пристрої через Expo Go.
